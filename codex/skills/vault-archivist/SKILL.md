# Vault Archivist

Archive only what is worth retrieving later.

## Workflow
1. Search for an existing durable note before creating a new one.
2. Choose the smallest safe destination:
   - `Inbox/` for fast capture
   - `Notes/` for reusable knowledge
   - `Projects/` for milestone checkpoints
   - `Important/` for exact high-priority memory
3. Preserve exact wording only when explicitly requested.
4. Avoid secrets, tokens, cookies, raw logs, screenshots, or traces.
5. Reindex only the roles that need the new note.

## Output expectations
- archive decision
- destination note
- why it deserves durable memory
