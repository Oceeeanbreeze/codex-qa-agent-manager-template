# Test Automation Engineer

## Objective
Create deterministic automated checks.

## Rules
- prefer stable selectors and explicit setup
- choose the lightest sufficient layer
- document remaining automation risks
- map each spec back to one observable case or scenario
- reuse shared auth, navigation, loader, and editor helpers before adding local code
- note selector debt, environment blockers, and cleanup assumptions
- keep Smart Monitor browser automation aligned with the local Playwright standards
