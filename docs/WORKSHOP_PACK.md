# Workshop Pack

## Goal
Provide a short workshop format for onboarding a QA team to the agent system.

## Audience
- testers
- automation engineers
- QA leads

## 60-minute workshop outline
1. `10 min` System shape and operator model
2. `10 min` Runtime gates: `doctor`, `health`, constrained-host fallback
3. `10 min` Smart Monitor QA flow from analysis to automation
4. `10 min` Atomic testcase drafting exercise
5. `10 min` Playwright review exercise
6. `10 min` Q&A and adoption decisions

## Demo path
- open `docs/FIRST_HOUR_RUNBOOK.md`
- open `docs/SMART_MONITOR_QA_AUTOMATION_PLAYBOOK.md`
- show `codex/SKILL_ROUTING.md`
- show one role doc such as `codex/agents/test-strategist.md`
- show `docs/PLAYWRIGHT_SMART_MONITOR_STANDARDS.md`

## Suggested exercises
- turn one vague request into an atomic testcase
- classify one task into the right test layer
- review one unstable Playwright test for selector and wait issues
- identify one environment blocker versus one product bug

## Success criteria
- the team can name the default role chain
- the team can explain when to use `doctor` and `health`
- the team can produce one atomic testcase correctly
- the team can explain why browser validation is or is not needed
- the team knows where to find the main Smart Monitor QA playbook
